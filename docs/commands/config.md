# Команды Config ⚙️

Команды для просмотра и изменения пользовательской конфигурации `dodocli`. Настройки хранятся в файле `~/dodocli/settings.json`.

---

## 👀 Просмотр настроек (View)

Показывает текущие значения конфигурации. Секреты маскируются, поэтому вывод безопасно прикладывать в обращения и issue.

```bash
dodo config view
```

### Пример вывода

```
email:        user@dodobrands.io
duty-api-key: 6e88****
auto-update:  true
```

Значения, которые ещё не заданы, отображаются как `(not set)`.

---

## ✏️ Изменение настройки (Set)

Устанавливает значение конфигурации по ключу.

```bash
dodo config set <key> <value>
```

### Поддерживаемые ключи

| Ключ | Значения | Описание |
| :--- | :--- | :--- |
| `auto-update` | `true` \| `false` | Автоматически устанавливать новые версии `dodocli`. По умолчанию: `true`. |

### Примеры

Отключить автообновление (получать только уведомления о новых версиях):
```bash
dodo config set auto-update false
```

Включить автообновление обратно:
```bash
dodo config set auto-update true
```

---

## 🧐 Как это работает

1. **Хранение**: Настройки сохраняются в `~/dodocli/settings.json`. Команда `set` дописывает/перезаписывает соответствующее поле, остальные значения не затрагиваются.
2. **Автообновление**: Параметр `auto-update` управляет поведением проверки версий:
   - `true` — при появлении новой версии (включая патч) `dodocli` обновляется автоматически на следующем запуске;
   - `false` — обновление не выполняется, вместо этого выводится уведомление с предложением запустить `dodo upgrade`.
3. **Глобальный override**: Если `dodocli` собран с ldflag `disableAutoUpdate=true`, автообновление принудительно отключено независимо от значения `auto-update` в настройках.

**Важно:** значение по умолчанию для `auto-update` — `true`. Если поле отсутствует в `settings.json`, автообновление считается включённым.
